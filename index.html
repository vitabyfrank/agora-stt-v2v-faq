시작하기Getting Started
STT(V2V) 서비스를 사용하려면 어떻게 해야 하나요? How do I get started with the STT(V2V) service?
먼저 Agora Console에서 Conversational AI Engine을 활성화해야 합니다. 활성화하지 않으면 API 호출 시 권한 오류가 발생합니다. 활성화 방법: Agora Console → 프로젝트 선택 → Usage → Conversational AI Engine → 활성화(Enable) You must first enable Conversational AI Engine in the Agora Console. Without activation, API calls will return permission errors. How to enable: Agora Console → Select Project → Usage → Conversational AI Engine → Enable
사용량Usage
사용량은 어디서 확인할 수 있나요? Where can I check my usage?
STT(V2V) 사용량은 Agora Console의 Conversational AI Engine 항목에서 직접 확인하실 수 있습니다.
STT(V2V) 솔루션과 Conversational AI Engine의 사용량(usage)을 Console에서 분리하여 확인할 수 있는 기능은 현재 준비 중에 있습니다. Agora Console Usage → STT (V2V) usage is currently available in the Conversational AI Engine section of the Agora Console.

자주 묻는 질문Frequently Asked Questions
이 API는 여러 언어를 지원하나요? Does this API support multiple languages?
네, 지원합니다. 다만 각 언어별로 별도의 API(에이전트)를 시작해야 합니다. Yes, but you need to start a separate agent for each language.
하나의 채널에서 여러 에이전트를 동시에 사용할 수 있나요? Can I run multiple agents in a single channel at the same time?
가능합니다. 각 에이전트에 고유한 agent name을 부여하면 동일 채널에서 병렬 운영됩니다. Yes. Assign a unique agent name to each agent and they can operate in parallel within the same channel.
통화 중 에이전트가 응답하지 않습니다. The agent is not responding during a call.
아래 트러블슈팅 템플릿을 작성하여 지원팀에 공유해 주세요. Please fill out the troubleshooting template below and share with support.
API 응답은 200(성공)인데 에이전트가 동작하지 않습니다. API returns 200 (success) but the agent is not working.
RTC 토큰 서버와 Agent 서버는 분리되어 운영됩니다. STT 서버에서 요청을 정상적으로 수신하여 200 응답을 반환하더라도, RTC 토큰에 문제가 있을 경우 에이전트는 채널에 참여하지 못합니다. 확인 사항: 토큰이 agent_rtc_uid에 맞게 발급되었는지, 토큰이 만료되지 않았는지 확인해 주세요. The RTC token server and Agent server operate independently. Even if the STT server accepts the request and returns a 200 response, the agent will not function properly if there is an issue with the RTC token. Check: Verify that the token was generated for the correct agent_rtc_uid and that it has not expired.
하나의 worker에서 여러 언어를 동시에 번역할 수 있나요? Can a single worker translate multiple languages at the same time?
아니요. STT(V2V) 서비스는 품질과 안정성을 위해 worker 1개당 1개 번역 언어만 지원하도록 설계되어 있습니다. 다중 언어 자막이 필요한 경우, 언어별로 worker를 분리하여 각각 구성해야 합니다. 예시:
"name": "translation_en" → 한국어 → 영어
"name": "translation_zh" → 한국어 → 중국어 No. The STT(V2V) service is designed to support only one translation language per worker for quality and stability. For multi-language subtitles, configure separate workers for each target language. Example:
"name": "translation_en" → Korean → English
"name": "translation_zh" → Korean → Chinese
agent_rtc_uid와 remote_rtc_uids의 차이점은 무엇인가요? What is the difference between agent_rtc_uid and remote_rtc_uids?
agent_rtc_uid — 에이전트(번역 봇)의 UID. 채널에 참여할 에이전트의 고유 ID를 지정합니다.
remote_rtc_uids — 번역 대상 유저의 UID 목록. 에이전트가 음성을 수신할 실제 사용자의 UID를 지정합니다. 흔한 실수: remote_rtc_uids에 존재하지 않는 UID를 넣거나, 에이전트 자신의 UID를 넣는 경우가 있습니다. 반드시 채널에 참여 중인 유저의 UID를 지정해 주세요. agent_rtc_uid — The agent's (translation bot) UID. Specifies the unique ID for the agent joining the channel.
remote_rtc_uids — List of target user UIDs. Specifies the actual users whose audio the agent will receive and translate. Common mistake: Using a non-existent UID or the agent's own UID in remote_rtc_uids. Always use the UID of a user who is active in the channel.
에이전트 토큰은 어떻게 발급해야 하나요? How should I generate the agent token?
token은 반드시 agent_rtc_uid에 해당하는 에이전트의 토큰으로 발급받아야 합니다. 예: agent_rtc_uid가 "1002"이면 → UID 1002에 대한 토큰을 생성 주의: 유저의 토큰을 에이전트 토큰 자리에 사용하면 오류가 발생합니다. The token must be generated for the agent_rtc_uid. e.g. agent_rtc_uid is "1002" → generate token for UID 1002 Note: Using a user's token in place of the agent's token will result in an error.
동일 채널에서 여러 언어로 번역 에이전트를 시작하면 오류가 발생합니다. Starting multiple translation agents in the same channel returns a TaskConflict error.
각 에이전트의 name 필드에 고유한 값을 사용해야 합니다. 동일한 name으로 두 번째 에이전트를 시작하면 기존 에이전트와 충돌하여 409 TaskConflict 오류가 발생합니다. 해결 방법: 언어별로 고유한 name을 부여하세요. "name": "agora_translation_test_id_en" → 영어 Agent
"name": "agora_translation_test_id_zh" → 중국어 Agent Each agent must have a unique name field. Starting a second agent with the same name causes a conflict with the existing agent, resulting in a 409 TaskConflict error. Solution: Assign a unique name per language. "name": "agora_translation_test_id_en" → English Agent
"name": "agora_translation_test_id_zh" → Chinese Agent
